# [FSx for Lustre](https://aws.amazon.com/ko/fsx/lustre/)

## <img src = "https://github.com/LeeWooJung/AWS-SAA-C03/assets/31682438/e3936f24-4682-4609-95c5-515b7254688b" width = "25" height = "25"> FSx for Lustre

* Amazon FSx for Lustre는 **고성능 컴퓨팅(HPC), 머신 러닝, 빅데이터 분석, 미디어 처리와 같은 데이터 집약적인 워크로드를 위한 완전 관리형 파일 시스템**.  
* Lustre 파일 시스템은 높은 처리량과 낮은 지연 시간을 제공하여 대규모 데이터를 빠르게 처리할 수 있도록 설계됨.

## 특징

* **고성능**  
    * Amazon FSx for Lustre는 초당 수백 기가바이트의 처리량과 **수백만 IOPS**(초당 입출력 작업)를 제공하여 대규모 데이터 처리에 최적화되어 있음.  
    * 데이터 집약적인 애플리케이션에서 뛰어난 성능을 발휘함.

* **완전 관리형**  
    * Amazon FSx for Lustre는 AWS에서 **완전히 관리**되므로 사용자는 파일 시스템의 구축, 유지 관리, 백업, 패치 등의 관리 작업에 신경 쓸 필요가 없음.  
    * AWS 콘솔을 통해 쉽게 파일 시스템을 생성하고 관리할 수 있음.

* **확장성**  
    * 필요에 따라 파일 시스템의 크기와 성능을 쉽게 확장할 수 있음.  
    * 용량과 성능 요구사항에 맞게 파일 시스템을 크기 조정할 수 있어 다양한 워크로드를 처리하는 데 유연하게 대응할 수 있음.

* **POSIX 호환**  
    * Lustre 파일 시스템은 **POSIX 호환 파일 시스템**으로, 다양한 운영 체제와의 호환성을 제공함.  
    * Linux 및 Unix 기반 애플리케이션에서 쉽게 사용할 수 있음.

* **EC2 및 S3 통합**  
    * **Amazon EC2 인스턴스**에서 Lustre 파일 시스템을 마운트하여 고성능 스토리지를 사용할 수 있으며, **Amazon S3와의 통합**을 통해 S3 버킷의 데이터를 Lustre 파일 시스템에 로드하거나 Lustre 파일 시스템의 데이터를 S3에 저장할 수 있음.

* **고가용성 및 내구성**  
    * 파일 시스템 데이터는 여러 AWS 가용 영역(AZ)에 걸쳐 복제되어 **높은 가용성과 내구성**을 제공함.  
    * 장애 발생 시에도 데이터 접근이 가능함.

## Storage Option

Amazon FSx for Lustre는 다양한 스토리지 옵션을 제공하여 사용자의 성능 및 비용 요구사항에 맞게 파일 시스템을 구성할 수 있게 했음. 

* **SSD**(솔리드 스테이트 드라이브)  
    * 높은 IOPS와 낮은 지연 시간을 제공하여 고성능 컴퓨팅, 머신 러닝, 미디어 처리와 같은 데이터 집약적인 애플리케이션에 적합함.  
    * SSD 스토리지는 **높은 처리량과 빠른 응답 시간을 요구하는 워크로드에 최적**임.

* **HDD**(하드 디스크 드라이브)  
    * 대용량 데이터를 경제적으로 저장할 수 있는 옵션으로, 빅데이터 분석, 백업 및 아카이빙 등 대규모 데이터 세트를 저장하면서 **비용을 절감**하려는 경우에 적합함.  
    * HDD 스토리지는 상대적으로 낮은 비용으로 대용량 스토리지를 제공함.

## Seamless integration with S3

Amazon FSx for Lustre는 **Amazon S3와의 원활한 통합**을 제공하여 S3 버킷의 데이터를 쉽게 액세스하고 사용할 수 있게 했음. 

* **데이터 가져오기(Import)**  
    * S3 버킷에 저장된 데이터를 FSx for Lustre 파일 시스템으로 가져와 고성능 컴퓨팅 작업을 수행할 수 있음.  
    * 이를 통해 대규모 데이터 세트를 S3에서 FSx for Lustre로 쉽게 이동하여 고속 처리할 수 있음.

* **데이터 내보내기(Export)**  
    * FSx for Lustre 파일 시스템에서 생성된 데이터를 S3 버킷으로 내보내어 **장기 저장, 분석, 백업 및 아카이빙**할 수 있음.  
    * 이를 통해 고성능 처리가 완료된 데이터를 S3에 안전하게 저장할 수 있음.

* **자동 동기화**  
    * FSx for Lustre와 S3 간의 **데이터 동기화 작업**을 자동화하여, S3에 저장된 데이터를 최신 상태로 유지할 수 있음.  
    * 이를 통해 데이터 일관성을 유지하면서 작업을 수행할 수 있음.

## On-premise에서의 사용

Amazon FSx for Lustre는 온프레미스 환경에서도 사용할 수 있으며, 이를 통해 하이브리드 클라우드 환경을 구축할 수 있음.

* **AWS Direct Connect**  
    * AWS Direct Connect를 사용하여 **온프레미스 데이터 센터와 AWS를 고속 전용 네트워크로 연결**할 수 있음.  
    * 이를 통해 온프레미스 애플리케이션이 FSx for Lustre 파일 시스템을 마치 로컬 스토리지처럼 사용할 수 있음.

* **하이브리드 클라우드 워크로드**  
    * 온프레미스와 클라우드를 병행하여 사용하는 하이브리드 클라우드 워크로드에 적합함.  
    * 온프레미스에서 데이터 처리를 시작하고, 필요에 따라 클라우드의 FSx for Lustre를 활용하여 확장된 처리 용량을 제공할 수 있음.

* **데이터 백업 및 복구**  
    * 온프레미스 데이터의 백업 및 복구 솔루션으로 FSx for Lustre를 사용할 수 있음.  
    * 온프레미스 데이터를 FSx for Lustre로 백업하고, 필요 시 클라우드에서 데이터를 복구하여 사용할 수 있음.

## File system 배포 옵션

Amazon FSx for Lustre는 다양한 파일 시스템 배포 옵션을 제공하여 사용자의 요구에 맞게 파일 시스템을 구성할 수 있게 했음.

### Scratch File System

* Scratch File System은 **일시적인 데이터 저장 및 고성능 임시 작업을 위해 설계**되었음.  
* 주로 단기 데이터 처리 작업에 사용되며, **내구성을 제공하지 않음**.
* **특징**
    * **고성능**  
    높은 처리량과 저지연 성능을 제공하여 빠른 데이터 처리 작업에 적합함.
    * **저비용** 
    내구성 없이 비용 효율적으로 설계되어 일시적인 데이터 저장 및 고성능 작업에 적합함.
    * **비영구적 데이터 저장**  
    인스턴스가 종료되거나 시스템이 재부팅되면 데이터가 유지되지 않음.

Scratch File System은 주로 과학적 시뮬레이션, 미디어 렌더링, 머신 러닝 모델 훈련과 같은 일시적인 고성능 컴퓨팅 작업에 사용됨.

### Persistent File System
* Persistent File System은 **장기 데이터 저장 및 내구성**이 필요한 워크로드를 위해 설계되었음.  
* 데이터가 파일 시스템에 **영구적으로 저장**되며, **고가용성과 내구성**을 제공함.

* **특징**
    * **내구성**  
    데이터를 같은 가용 영역(AZ)에 복제하여 높은 내구성을 제공함.
    * **고가용성**  
    장애 발생 시에도 데이터 접근이 가능하도록 설계되었음.
    * **장기 데이터 저장**  
    인스턴스 종료나 시스템 재부팅 시에도 **데이터가 유지**됨.

Persistent File System은 **장기 데이터 저장이 필요한 엔터프라이즈 애플리케이션, 데이터 분석, 백업 및 복구 워크로드에 적합**함.

### Data Repository Integration (DRI)

* Data Repository Integration (DRI)은 Amazon S3와의 통합을 통해 대규모 데이터를 쉽게 관리하고 처리할 수 있도록 설계되었음.

* **특징**
    * **S3 데이터 접근**  
    S3 버킷에 저장된 데이터를 FSx for Lustre 파일 시스템에서 직접 액세스하고 사용할 수 있음.
    * **데이터 이동 자동화**  
    S3와 FSx for Lustre 간의 데이터 이동을 자동화하여 데이터 일관성을 유지함.
    * **고성능 데이터 처리**  
    S3의 데이터를 고성능 파일 시스템에서 처리하여 대규모 데이터 분석, 머신 러닝, 미디어 처리 작업에 적합함.

DRI를 통해 S3에 저장된 데이터를 FSx for Lustre 파일 시스템으로 쉽게 가져와서 고성능으로 처리할 수 있음.

## Use case

* **고성능 컴퓨팅(HPC)**  
과학 연구, 공학 시뮬레이션, 금융 모델링 등 대규모 데이터를 빠르게 처리해야 하는 HPC 워크로드에 적합함.

* **빅데이터 분석**  
Apache Hadoop, Apache Spark 등의 빅데이터 프레임워크와 통합하여 대규모 데이터 세트를 분석하는 데 사용됨.

* **미디어 처리**  
영상 렌더링, 비디오 편집 등 고성능 스토리지가 필요한 미디어 처리 워크로드에 적합함.

* **머신 러닝**  
대규모 데이터 세트를 학습시키는 머신 러닝 모델 트레이닝에 적합하며, 빠른 데이터 접근을 통해 모델 트레이닝 시간을 단축할 수 있음.
